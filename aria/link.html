link – inter-document relationship metadata # T

The link element represents metadata that expresses inter-document relationships.

Content model #

link = empty & link.attrs

Attribute model #

link.attrs = common.attrs & href & rel & hreflang? & media? & type? & sizes?

Element-specific attributes #

href = uri
A URL that provides the destination of the link.
rel = tokens
A list of tokens that specify the relationship between the document containing the link and the destination indicated by the link.
hreflang = langcode
The language of the destination of the link.
media = mediaquery
The media for which the destination of the link was designed.
type = mimetype
The MIME type of the destination of the link.
sizes = "any" | sizes
The sizes of icons for visual media.

Tag omission #

The link element is a void element. A link element must have a start tag but must not have an end tag.

Permitted contexts #

  • common.elem.metadata = link | style | meta.name | meta.http-equiv.refresh | meta.http-equiv.default-style | meta.http-equiv.content-language | meta.charset | meta.http-equiv.content-type | script | noscript.elem.head | command
  • noscript.elem.head = (link | meta.http-equiv.default-style | meta.http-equiv.refresh | style)*

Typical default display properties #

link {
display: none; }
